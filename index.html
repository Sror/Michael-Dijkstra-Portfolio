Michael Dijkstra.

User interface designer and front-end developer.

Passionate about web standards (making websites accessible to everyone), HTML5 and CSS3 (the future of the web) and Responsive Web Design (designing for all devices). Love working with Ruby on Rails and building web applications that help real people.

Managing all of this from home in Sydney, Australia.

I believe

  • Ideas are developed on paper with a thick black sharpie.
  • Prototyping and design should happen in code, in the browser.
  • Websites should respond to the user, not be static.
  • A useful interface is more important than a beautiful interface.
  • Copywriting is design and needs to be included from the beginning.

Skills

  • Hand code clean, well-structured, semantic HTML5 markup.
  • Code clean, well-structured CSS3 that's cross browser compatible.
  • Prefer to use Sass to create cleaner, more awesome CSS.
  • Employ the Responsive Web Design principles.
  • Script clean and unobtrusive Javascript (jQuery).
  • Build sites with solid on-site SEO.
  • Develop for Content Management Systems (Wordpress, ExpressionEngine).
  • Build web applications using Ruby on Rails.
  • Manage version control with Git.
